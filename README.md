# subscriptions-service

Тестовое задание

**Задача:** спроектировать и реализовать REST-сервис для агрегации данных об онлайн-подписках пользователей.

**Реализовано:**
- CRUDL-операции над подписками (service_name, price, user_id, start_date, end_date)
- Эндпоинт подсчёта суммарной стоимости подписок за период с фильтрацией по пользователю и названию сервиса — стоимость считается пропорционально количеству полных месяцев пересечения с запрошенным периодом
- PostgreSQL, миграции через goose (применяются при старте сервиса)
- Логирование через `slog`
- Конфигурация через `.env`
- Swagger-документация
- Запуск через Docker Compose

## Запуск

```bash
cp .env.example .env
# заполнить DB_PASSWORD и при необходимости изменить остальные значения
docker compose up --build
```

Сервис будет доступен по адресу `http://localhost:8888`.  
Swagger UI: `http://localhost:8888/swagger/index.html`

## Переменные окружения

| Переменная    | Описание                        | Пример              |
|---------------|---------------------------------|---------------------|
| `APP_PORT`    | Порт HTTP-сервера               | `8888`              |
| `DB_HOST`     | Хост Postgres                   | `localhost`         |
| `DB_PORT`     | Порт Postgres                   | `5432`              |
| `DB_USER`     | Пользователь Postgres           | `postgres`          |
| `DB_PASSWORD` | Пароль Postgres                 | `secret`            |
| `DB_NAME`     | Название базы данных            | `subscriptions_db`  |
| `LOG_LEVEL`   | Уровень логирования (debug/info/warn) | `info`        |

## Эндпоинты

| Метод    | Путь                     | Описание                                       |
|----------|--------------------------|------------------------------------------------|
| `POST`   | `/subscriptions`         | Создать подписку                               |
| `GET`    | `/subscriptions`         | Получить список подписок (фильтр по user/name) |
| `GET`    | `/subscriptions/{id}`    | Получить подписку по ID                        |
| `PUT`    | `/subscriptions/{id}`    | Обновить подписку                              |
| `DELETE` | `/subscriptions/{id}`    | Удалить подписку                               |
| `GET`    | `/subscriptions/total`   | Сумма расходов пользователя за период          |

Даты передаются в формате `MM-YYYY`. Цены — целые числа.

Полные схемы запросов и ответов — в Swagger UI.
